# 1.1.POST/api/v2/processo/enviar-documento-para-assinar

El objetivo del método es permitir a los Clientes, a través de la API, enviar procesos con uno o más documentos para firmar.

A diferencia de la aplicación ArqSign, que permite agrupar los archivos del proceso, en este servicio, al enviar el proceso con más de un documento, el servicio los considera como **una colección de archivos y no serán agrupados en un único archivo**. Es decir, no es posible agrupar archivos mediante el servicio de envío de procesos de documentos para firmar.

El usuario debe proporcionar los datos del proceso y de los destinatarios participantes en el proceso de firma.

## **Autenticaciones y permisos**

1. Para autenticarse en la API de ArqSign, el usuario debe proporcionar la AppKey válida de la cuenta que está enviando los documentos para firmas.
2. Solo las cuentas con **estado Activo** y con **permiso de Integración ArqSign** pueden enviar procesos mediante integración con ArqSign.
3. La cuenta debe contar con créditos de Envíos, SMS y/o WhatsApp, según la necesidad.
4. Si la cuenta no dispone de crédito, el sistema devuelve el mensaje:\
   &#xNAN;_"Saldo insuficiente de créditos de Envíos, WhatsApp o SMS."_

## **Solicitud: Parámetros**

**Orientaciones:**

Cuando el campo requerido sea “Sim” = Siempre requerido.

Cuando el campo requerido sea “Não” = Información Opcional.

Cuando el campo requerido sea “Talvez” = En algunos casos será requerido. Para conocer estos casos, consultar la descripción del tema según el número de referencia de la línea en la tabla.

Cuando el campo requerido sea “Não” = Información Opcional.

## **Descripción de los parámetros del JSON:**

### **idResponsavel**

Parámetro **obligatorio** que indica el ID del responsable del proceso. Esta información será considerada como el remitente del proceso. Si el usuario autenticado está relacionado con este ID, podrá visualizar todos los documentos en el menú "Enviados".\
Se debe proporcionar el ID de un usuario activo que pertenezca a la cuenta.

### **nomeProcesso**

Parámetro **obligatorio** que especifica el nombre del proceso, permitiendo hasta 250 caracteres.

### **idPasta**

Parámetro obligatorio que indica el ID de la carpeta del proceso.\
Se debe proporcionar un ID de carpeta que pertenezca a la cuenta.

### **configuracoesAvancada**

En esta sección del JSON, el usuario puede enviar configuraciones avanzadas para el proceso, como tiempo de expiración, aviso previo a la expiración, frecuencia de los avisos, generación de QRCode, y obligatoriedad de lectura de los documentos.

Estas configuraciones son **opcionales**. **Si no se envían** o se envían como nulas, el sistema u**sará las configuraciones predeterminadas de la cuenta**.

**tempoExpiracaoDocumentoDias**

Parámetro **opcional** que especifica el tiempo que el proceso estará disponible para firma, hasta que todos los signatarios firmen.\
Valores permitidos: entre {1} y {999}.

Si no se envía o se envía como nulo o 0, el sistema usará las configuraciones predeterminadas de la cuenta.

**avisoAntesExpiracaoDocumentoDias**

Parámetro opcional que define el tiempo antes de que el documento expire para que la aplicación comience a enviar avisos.\
Valores permitidos: entre {1} y {99}.

Si no se envía o se envía como nulo o 0, el sistema usará las configuraciones predeterminadas de la cuenta.

**frequenciaLembretesDias**

Parámetro **opcional** que especifica la frecuencia de los avisos de firma.\
Valores permitidos: entre {1} y {999}.\
Si no se envía o se envía como nulo o 0, el sistema usará las configuraciones predeterminadas de la cuenta.

**gerarQrCode**

Parámetro **opcional** que indica si se debe generar un QRCode del documento firmado al finalizar el proceso.\
Valores permitidos: 1 = true, 0 = false.\
Si no se envía o se envía como nulo o 0, el sistema usará las configuraciones predeterminadas de la cuenta.

**obrigarLeitura**

Parámetro **opcional** que establece si es obligatorio leer los documentos antes de firmarlos.\
Si el proceso tiene más de un documento, esta configuración obligará al signatario a abrir cada documento antes de habilitar el botón "Firmar".\
Valores permitidos: 1 = true, 0 = false.\
Si no se envía o se envía como nulo o 0, el sistema usará las configuraciones predeterminadas de la cuenta.

### **renovacaoMeses**

Parámetro **opcional** que indica el tiempo, en meses, para definir cuándo el proceso en cuestión necesita ser renovado.

### **mensagemPadrao**

Parámetro opcional que especifica el mensaje predeterminado que se enviará a todos los destinatarios definidos como firmantes del proceso.\
Si la colección de datos **"mensagemPadrao" no se envía**, el sistema utiliza los valores predeterminados:                                                                                                                                                 **Título=** ArqSign: \[Nombre del documento indicado en el parámetro "nome"]                                        **Texto=** Usted recibió el(los) documento(s): \[Nombre del documento indicado en el parámetro "nome"] para firmas.

**titulo**\
Parámetro **opcional** que especifica el título del mensaje que se enviará a los firmantes.

**texto**\
Parámetro **opcional** que especifica el texto del mensaje que se enviará a los firmantes.

### **usarOrdemAssinatura**

Parámetro **obligatorio** que indica si se usará o no un orden de firma para los firmantes.\
Valores permitidos: 1 = true, 0 = false.

a. Si el parámetro **usarOrdemAssinatura** se envía como **false**, el sistema ignora el valor del parámetro **"ordemAssinatura"** de todos los destinatarios con acción de Firmar Online y envía el proceso a todos los destinatarios con el orden 1.

b. Si el parámetro **usarOrdemAssinatura** se envía como **true**, el sistema valida el parámetro **"ordemAssinatura"** de todos los destinatarios con acción de Firmar Online. Es decir, el valor de **"ordemAssinatura"** no puede ser cero ni nulo.

En este caso, el sistema envía el proceso solo a los destinatarios con orden 1.

### **usarPosicaoAssinaturaAutomatica**

Parámetro **obligatorio** que indica si el proceso incluirá una página de firma al final de cada archivo, con la posición de las firmas asignada automáticamente.\
Valores permitidos: 1 = true, 0 = false.

a. Si el parámetro **usarPosicaoAssinaturaAutomatica** se envía como **true**, la aplicación **incluye una página al final de cada archivo con la posición de la firma asignada automáticamente**, según el tipo de firmante:

* **assinarComo = 1 (Persona física)**: Se incluye una página con la posición de firma automática para Persona Física (PF).
* **assinarComo = 2 (Persona jurídica)**: Se incluye una página con la posición de firma automática para Persona Jurídica (PJ).
* **assinarComo = 3 (Persona física y jurídica)**: Se incluye una página con posiciones de firma automática para ambos tipos (PF y PJ).

Cuando el parámetro **“usarPosicaoAssinaturaAutomatica”** se envíe como true, no es necesario enviar los parámetros **definirPosicaoAssinaturaManual**. Sin embargo, si en el JSON se incluye este parámetro para algún firmante, la información será ignorada en este caso.

b. Cuando el parámetro **usarPosicaoAssinaturaAutomatica** se envíe como **false**, significa que la ubicación de las firmas **para cada firmante** deberá ser informada manualmente **para cada archivo del proceso** en cuestión, siendo obligatorio enviar los datos del parámetro **definirPosicaoAssinaturaManual**.

{% hint style="danger" %}
<mark style="color:red;">**Importante:**</mark> <mark style="color:red;"></mark><mark style="color:red;">En el JSON, todos los destinatarios con acción de firmar en línea deben tener la ubicación de la representación visual definida automáticamente por la aplicación o, de forma alternativa, debe enviarse manualmente para todos en el JSON. No puede haber, en el mismo proceso, algunos destinatarios con ubicación automática y otros sin ubicación automática.</mark>
{% endhint %}

### **Destinatarios**

**idTipoAcao**

Parâmetro obrigatório que define la acción del destinatario respecto al documento. Actualmente existen dos opciones **1 - Firmar en línea** o **2 - Recibir una copia**.\
Es permitido informar 1 - Firmar en línea o 2 - Recibir una copia.

**a.** Es obligatorio informar para el proceso de firmas, al menos un destinatario con acción de **“Firmar en línea”**.

**ordenAssinatura**

Parámetro que define el orden de firma del destinatario en cuestión.

**a.** Cuando el parámetro **“usarOrdemDeAssinatura”** esté siendo enviado como **True = 1** y el destinatario en cuestión posea el parámetro **idTipoAcao = 1 (Firmar en línea)**, entonces esta información es obligatoria.

El parámetro **ordenAssinatura** no debe ser enviado en el JSON o será desconsiderado cuando:               - **idTipoAcao = 2 (Recibir una copia)**.                                                                                                                          - **usarOrdemDeAssinatura = 0 (fase)** y destinatarios con **idTipoAcao = 1 (Firmar en línea)**.

**b.** La aplicación **no permite el mismo destinatario en la misma orden de firma**. Para verificar si el destinatario es el mismo, verifica si el correo electrónico o el teléfono utilizados como forma de envío para el documento se repiten en la misma orden de firma.

**nome**

Parámetro **obligatorio** que informa el nombre del destinatario que firmará el documento o recibirá una copia.

**a.** El sistema valida la obligatoriedad y el tamaño del nombre del destinatario.

**email**

Parámetro que informa el correo electrónico donde el destinatario recibirá el documento.

**Formato:** Varchar(150)

**Requerido: El correo electrónico o el teléfono son obligatorios.**

**Validación:**

**a.** El sistema valida la **obligatoriedad** del correo electrónico o teléfono.\
Cuando no se envíe ninguno de los dos, la aplicación valida los datos mostrando el mensaje: El parámetro correo electrónico o teléfono es obligatorio.

**b.** Solo es permitido **correo electrónico válido.**

**c.** Solo **es permitido informar correo electrónico o teléfono.**\
Cuando el usuario informe los dos campos, el sistema retorna el mensaje: Solo es posible enviar el proceso a un correo electrónico o teléfono. Elija una forma de envío.

**telefone**

Parámetro que informa el número de teléfono donde el destinatario recibirá el documento.\
Enviando el número de teléfono, significa que el documento será enviado por **WhatsApp.**

**a.** El sistema valida la **obligatoriedad** del correo electrónico o teléfono.\
Cuando no se envíe ninguno de los dos, la aplicación valida los datos mostrando el mensaje: El parámetro **correo electrónico o teléfono** es obligatorio.

**b.** El sistema **valida** el teléfono informado.

**c.** Solo **es permitido informar correo electrónico o teléfono**.\
Cuando el usuario informe los dos campos, el sistema retorna el mensaje: Solo es posible enviar el proceso a un correo electrónico o teléfono. Elija una forma de envío.

**d.** Solo es posible enviar proceso por WhatsApp, cuando el destinatario cumpla con uno de los requisitos a continuación:                                                                                                                                             **1.** Para firmar el documento electrónicamente idTipoAcao **= 1 (Firmar en línea) y idTipoAssinatura = 1 (Firma Electrónica).**                                                                                **2.** Para recibir una copia: **idTipoAcao = 2 (Recibir una copia).**

### **assinarOnline**

Esta parte do JSON debe enviarse solo para los destinatarios con idTipoAcao = 1 (Firmar en línea). Si se envía para un destinatario con idTipoAcao = 2 (Recibir una copia), estos datos serán ignorados.

**assinarComo**

Parámetro **obligatorio** que indica si el signatario, con el idTipoAcao = 1 (Firmar en línea), firmará como persona física, persona jurídica o ambas.\
Es permitido informar 1 = Persona Física, 2 = Persona Jurídica o 3 = Persona Física y Jurídica.

**papelPessoaFisica**\
Parámetro opcional para enviar los papeles del signatario en relación a la firma de persona física que realizará.\
a. El parámetro papelPessoaFisica es opcional, pero cuando se envíe, el sistema verifica si la cuenta posee el papel del signatario informado.\
El usuario podrá enviar null o no informar este parámetro en el JSON.

Si el papelPessoaFisica se envía para un signatario con el parámetro “assinarComo” = 2 (Persona Jurídica), entonces esta información es desconsiderada.

**papelPessoaJurídica**\
Parámetro opcional para enviar los papeles del signatario en relación a la firma de persona jurídica que realizará.\
a. El parámetro papelPessoaJurídica es opcional, pero cuando se envíe, el sistema verifica si la cuenta posee el papel del signatario informado.\
El usuario podrá enviar null o no informar estos parámetros en el JSON.

Si el papelPessoaJuridica se envía para un signatario con el parámetro “assinarComo” = 1 (Persona Física), entonces esta información es desconsiderada.

**seguranca**\
Esta parte del JSON es opcional, deberá enviarse solo si existe la necesidad de exigir al signatario, en cuestión, un código de seguridad para iniciar el proceso de firma.

**codigoSeguranca**\
Código de seguridad a ser exigido al signatario que irá firmar el documento. Esta información necesita ser numérica con 4 caracteres.

**idMeioEnvio**\
Parámetro para informar cómo el código de seguridad será enviado al signatario en cuestión.\
Este parámetro solo será exigido caso se envíe el parámetro codigoSeguranca y solo es permitido valores iguales a:\
1 = SMS (Solo Brasil),\
2 = WhatsApp,\
3 = Email,\
4 = No enviar.

El sistema desconsidera la información de este parámetro caso el codigoSeguranca = null.

**codigoSegurancaEmail**\
Correo electrónico donde el signatario deberá recibir el código de seguridad definido en el parámetro codigoSeguranca.\
a. El parámetro codigoSegurancaEmail es obligatorio cuando el parámetro idMeioEnvio sea igual a 3.

b. La aplicación valida si la información enviada es un correo electrónico.

Caso este parámetro sea enviado cuando idMeioEnvio <> 3, el sistema ignora la información.

El sistema desconsidera la información de este parámetro caso el codigoSeguranca = null.

**codigoSegurancaTelefone**\
Número de teléfono donde el signatario deberá recibir el código de seguridad definido en el parámetro codigoSeguranca.\
a. El parámetro codigoSegurancaTelefone es obligatorio cuando el parámetro idMeioEnvio sea igual a 1 o 2.

Cuando idMeioEnvio = 1 (SMS), solo se aceptarán números de Brasil.

El sistema valida el teléfono informado.

Caso este parámetro sea enviado cuando idMeioEnvio = 3 o 4, el sistema ignora la información.

El sistema desconsidera la información de este parámetro caso el codigoSeguranca = null.

**reenviarCodigo**\
Información que define si el signatario podrá reenviar el código de seguridad a su teléfono vía SMS o WhatsApp conforme la definición en el parámetro idMeioEnvio.\
Cuando no se envíe el parámetro reenviarCodigo en el JSON, el servicio considera como false.

Es permitido informar 1 = True o 0 = False.\
a. Este parámetro solo es obligatorio cuando el código de seguridad sea enviado por SMS o WhatsApp.

Caso este parámetro sea enviado cuando idMeioEnvio = 3 o 4, es desconsiderado.

**mensajePersonalizada**\
Esta parte del JSON es opcional y deberá enviarse solo si para el signatario en cuestión es necesario enviar un mensaje personalizado.\
Si se envía el título de la mensaje, será necesario enviar el texto y viceversa.

**titulo**\
Título de la mensaje personalizada a ser enviada para el signatario en cuestión. Los signatarios con mensaje personalizada no reciben la mensajePadrao.

a. El parámetro mensajePersonalizada no es obligatorio, pero cuando se envíe, el “titulo” la aplicación debe exigir el texto y viceversa.

**texto**\
Título de la mensaje personalizada a ser enviada para el signatario en cuestión. Los signatarios con mensaje personalizada no reciben la mensajePadrao.

a. El parámetro mensajePersonalizada no es obligatorio, pero cuando se envíe, el “titulo” la aplicación debe exigir el texto y viceversa.

**anexos**\
Esta parte del JSON es opcional y deberá enviarse solo si para el signatario en cuestión es necesario solicitar el upload de algún archivo en el momento de la firma.\
Si cualquier uno de los parámetros, a continuación, sea enviado, se exigirá los tres.

Si cualquier uno de los parámetros, a continuación, sea enviado, se exigirá los tres:\
**anexoDocumentoNome**\
Nombre del anexo que será solicitado al signatario en el momento de la firma de los documentos.\
Es permitido informar hasta 75 caracteres.

**anexoObrigatorio**\
Información para la aplicación exigir del signatario el upload del archivo en el acto de la firma del documento.\
Es permitido informar 1 = true o 0 = false.

**anexoExibirTodosDestinatarios**\
Información para la aplicación exibir el anexo que fue enviado por el signatario para todos los otros destinatarios con acción de firmar o no.\
Si el anexo a ser enviado exige un grado de sigilo, entonces esta información debe enviarse como false.\
Es permitido informar 1 = true o 0 = false.

**definirPosicaoAssinaturaManual**\
Esta parte del JSON es obligatoria solo si el parámetro usarPosicaoAssinaturaAutomatica es igual a false.\
Cuando el parámetro usarPosicaoAssinaturaAutomatica es enviado igual a false, significa que en el JSON deberá enviarse el posicionamiento manual de las firmas de todos los destinatarios con acción de Firmar en línea en cada archivo del proceso.\
O sea, es obligatorio definir la posición de la firma manualmente (definirPosicaoAssinaturaManual) para cada destinatario en cada archivo (documentoDeOrdem) conforme el valor definido en el parámetro assinarComo (1 = Persona Física o 2 = Persona Jurídica o 3 = Persona Física y Jurídica).

Cuando el parámetro usarPosicaoAssinaturaAutomatica es enviado como true, las informaciones de posicionamiento manual de la firma de todos los signatarios serán desconsideradas.\
Desta forma será incluida una página al final de cada documento con la posición de la firma de cada destinatario con acción de Firmar en línea.

El sistema valida si el posicionamiento manual de las firmas está en el límite de la página informada.\
El sistema valida las representaciones manuales informadas para los destinatarios no permitiendo la misma representación manual por tipo de firma en el mismo documento (parámetro documentoDeOrdem).

**documentoDeOrdem**\
Cuando el proceso de firma que está siendo enviado en el JSON posea más de un archivo para firma, este campo será usado para identificar el archivo al cual el posicionamiento de la firma que está siendo descrita en los parámetros a seguir se refiere.\
Si el proceso de firma posee solo un archivo a ser firmado, este campo se convierte en innecesario y será desconsiderado cuando enviado.\
a. Este parámetro es obligatorio siempre que en el proceso exista más de un archivo a ser firmado.\
b. El sistema valida si existe el documento de la orden informada en el parámetro “documentoDeOrdem”.

**pessoaFisica**\
Los parámetros abajo indicarán el posicionamiento de la firma de persona física para el signatario en cuestión, en el archivo identificado en documentoDeOrdem y serán obligatorios cuando los parámetros usarPosicaoAssinaturaAutomatica = false y assinarComo = 1 (PF) o 3 (PF y PJ).

Cuando el parámetro assinarComo = 2 (PJ), estos datos serán desconsiderados.\
Los parámetros abajo indicarán el posicionamiento de la firma de persona física para el signatario en cuestión, para el archivo identificado en documentoDeOrdem.

**pagina**\
Esta información es la página del archivo donde la posición de la firma será inserida. Es obligatoria cuando usarPosicaoAssinaturaAutomatica = false y assinarComo = 1 (PF) o 3 (PF y PJ).

a. Esta información no puede ser cero ni nula.

b. La aplicación valida si la página informada existe en el archivo en cuestión.

**altura**\
Altura en milímetros del tamaño de la caja donde la firma será aplicada. Es obligatoria cuando usarPosicaoAssinaturaAutomatica = false y assinarComo = 1 (PF) o 3 (PF y PJ).

a. Esta información no puede ser cero ni nula.

**largura**\
Anchura en milímetros del tamaño de la caja donde la firma será aplicada. Es obligatoria cuando usarPosicaoAssinaturaAutomatica = false y assinarComo = 1 (PF) o 3 (PF y PJ).

a. Esta información no puede ser cero ni nula.

**pessoaJuridica**\
Los parámetros abajo indicarán el posicionamiento de la firma de persona jurídica para el signatario en cuestión, en el archivo identificado en documentoDeOrdem y serán obligatorios cuando los parámetros usarPosicaoAssinaturaAutomatica = false y assinarComo = 2 (PJ) o 3 (PF y PJ).

Cuando el parámetro assinarComo = 1 (PF), estos datos serán desconsiderados.

Los parámetros abajo indicarán el posicionamiento de la firma de persona jurídica para el signatario en cuestión, para el archivo identificado en documentoDeOrdem.

**pagina**\
Esta información es la página del archivo donde la posición de la firma será inserida. Es obligatoria cuando usarPosicaoAssinaturaAutomatica = false y assinarComo = 2 (PJ) o 3 (PF y PJ).

a. Esta información no puede ser cero ni nula.

b. La aplicación valida si la página informada existe en el archivo en cuestión.

**altura**\
Altura en milímetros del tamaño de la caja donde la firma será aplicada. Es obligatoria cuando usarPosicaoAssinaturaAutomatica = false y assinarComo = 2 (PJ) o 3 (PF y PJ).

a. Esta información no puede ser cero ni nula.

**largura**\
Anchura en milímetros del tamaño de la caja donde la firma será aplicada. Es obligatoria cuando usarPosicaoAssinaturaAutomatica = false y assinarComo = 2 (PJ) o 3 (PF y PJ).

a. Esta información no puede ser cero ni nula.

**documentoDeOrdem**\
Cuando el proceso de firma que está siendo enviado en el JSON posea más de un archivo para firma, este campo será usado para identificar el archivo al cual el posicionamiento de la firma que está siendo descrita en los parámetros a seguir se refiere.\
Si el proceso de firma posee solo un archivo a ser firmado, este campo se convierte en innecesario y será desconsiderado cuando enviado.\
a. Este parámetro es obligatorio siempre que en el proceso exista más de un archivo a ser firmado.\
b. El sistema valida si existe el documento de la orden informada en el parámetro “documentoDeOrdem”.
